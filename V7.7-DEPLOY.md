## Regenerating the questions seed

If the survey structure changes (new questions added, wording refined, etc.),
regenerate the seed from a fresh xlsx:

```powershell
# Download the latest xlsx from Drive → File → Download → Microsoft Excel
# Save to data/responses.xlsx

python scripts\build_questions_metadata.py `
    --xlsx data\responses.xlsx `
    --out-sql data\questions_seed.sql `
    --out-json data\questions_manifest.json

cd worker
npx wrangler d1 execute circumsurvey --remote --file ..\data\questions_seed.sql
```

The `questions_manifest.json` is for human review — check tier assignments,
section categorization, and inferred option lists before re-importing.

## Schema reminder

The `questions` table was already created in v7.6.1's schema.sql. v7.7
only populates it. If you need to recreate from scratch:

```sql
CREATE TABLE questions (
  id         TEXT PRIMARY KEY,       -- stable slug, matches responses.question_id
  section    TEXT,                    -- e.g. "Demographics", "Observer Pathway"
  pathway    TEXT,                    -- all | intact | circumcised | restoring | observer | trans | intersex
  prompt     TEXT NOT NULL,           -- question wording verbatim from header
  subtitle   TEXT,                    -- secondary text after first punctuation break
  type       TEXT,                    -- scale_1_5 | single_select | multi_select | open_text
  opts_json  TEXT,                    -- JSON array of option labels, or NULL for open_text
  tier       INTEGER DEFAULT 3,       -- 1=curated, 2=browseable, 3=indexed
  col_idx    INTEGER                  -- 0-based original CSV column index
);
CREATE INDEX idx_questions_section ON questions(section);
CREATE INDEX idx_questions_pathway ON questions(pathway);
CREATE INDEX idx_questions_tier ON questions(tier);
```

## Rollback

If the questions seed causes problems:

```powershell
# Clear the questions table — rest of site still works with empty /api/questions
cd worker
npx wrangler d1 execute circumsurvey --remote --command "DELETE FROM questions"
```

Pages/Worker can also be reverted independently via `git revert HEAD && git push`.
