### `/api/geo` endpoint details

Answers queries like "how many respondents per US state?" for choropleth maps:

```
GET /api/geo?level=country                 → { locations: [{location: "USA", n: 317}, ...] }
GET /api/geo?level=us_state                → respondents by US state
GET /api/geo?level=us_state&by=pathway     → nested: {location, n, by_pathway: {intact: X, circ: Y}}
GET /api/geo?level=canada_province         → Canadian provinces
GET /api/geo?level=country&when=now        → use current location instead of birthplace (default)
```

This is what your future `/explore` or `/data` site will call to render maps.
The queries are cheap (scan a single indexed column in demographics).

## Cloudflare Web Analytics — ONE-CLICK SETUP

Short answer to your question: **yes, built in, one click, no code changes.**

1. Cloudflare dashboard → **Workers & Pages**
2. Select your `circumsurvey` Pages project
3. Go to **Metrics** tab
4. Click **Enable Web Analytics**

That's it. Cloudflare injects the privacy-first beacon on your next deployment.
Reports appear at `dash.cloudflare.com` → **Web Analytics**.

Features:
- Page views, unique visitors, top pages, referrers, country breakdowns
- Privacy-first: no cookies, no cross-site tracking, no PII, works without banner consent
- Free on all plans, no quota concerns at your traffic volume
- Core Web Vitals (LCP, FID, CLS) — real-user performance data

If you later want **custom events** ("how many people clicked the Six Pathways card",
"which Voices theme gets most interaction") — that's Workers Analytics Engine, a
separate beast. Not needed yet; easy to add later.

## Map visualizations — what's ready, what's next

Your DataStudio screenshot shows Google Maps heat points by birth location.
Rebuilding this in the findings/explore/data sites requires:

**Now (v7.6.1):**
- ✅ `/api/geo?level=us_state&by=pathway` returns the data
- ✅ D1 demographics table has indexed `country_born`, `us_state_born`, etc.

**v8.0 build (next):**
- Leaflet + React-Leaflet integration (~60 KB gzipped)
- Custom tile layer matching the Hello Bureau cream-paper aesthetic
  (options: Stadia Maps Stamen Terrain, CartoDB Positron, or a muted OSM style)
- Three map components:
  - **NA choropleth**: US states + Canadian provinces shaded by respondent count,
    click to see pathway breakdown tooltip
  - **World map**: proportional circles for international respondents
  - **Religion × geography grid**: small-multiples, one map per major tradition

Why Leaflet over Google Maps:
- Free forever, no API key, no quota
- Open data (OpenStreetMap contributors)
- Better styling control — can match your typography and palette exactly
- No phone-home to Google on every page view

D3 + TopoJSON is an alternative for pixel-perfect branded cartography
(what The NYT uses), but it's 2-3× the dev time. Leaflet is the pragmatic choice.

## Known limitations

**Questions metadata table is empty.** The `questions` table will populate in
v7.7 from `Master-Questions-v2.3.md`. `/api/questions` returns `[]` until then.
Nothing else depends on it.

**18 deep-pathway columns use `q###` slugs.** These are Observer/Trans/Intersex
questions not present in the Meta Tag Mapping sheet. They're fully loaded and
queryable — just less descriptive identifiers. Fix is to add them to the Meta
Tag Mapping sheet upstream, then re-run the loader.

**`source_row_idx` traceability.** Each respondent row has its original xlsx
row number saved. If we need to correlate back to the spreadsheet, it's there.
